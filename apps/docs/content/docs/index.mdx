## What Kira Is

Kira is a Zig-hosted compiler and bootstrap toolchain with:

- a frontend pipeline that parses and validates a broader Kira language surface
- a VM backend that executes bytecode
- an LLVM-native backend that emits host executables and shared libraries
- a hybrid runtime that runs bytecode and native code together
- a manifest-driven native library and autobinding system
- a managed toolchain flow for `kirac`, templates, and pinned LLVM bundles

## Current State

Kira is not yet documenting a full language/runtime product with every planned feature implemented.

What is real in this repo:

- `@Main`, `@Runtime`, and `@Native`
- a working VM path
- a working LLVM/native path
- a working hybrid path
- manifest-driven native library discovery
- Clang-driven autobinding generation
- generated Kira bindings for callbacks and Sokol
- corpus coverage for parser, semantics, VM, LLVM, hybrid, callbacks, and Sokol proofs

What you should not assume yet:

- dynamic linking as the documented first-class flow
- libffi-based runtime interop
- a stable Kira ABI for third-party binary ecosystems
- every frontend feature being runnable through `run` or `build`
